## Context

当前战斗场景 `battle_map.tscn` 已实现基础架构：
- **BattleCamera**: 触控相机控制（平移、缩放、网格选择信号）
- **GridMapSystem**: 网格地图管理和坐标转换
- **BattleHUD**: 单位信息面板和硬编码操作按钮（Move/Attack/Wait）

缺乏的关键交互组件：
- 用户点击格子后无视觉反馈指示选中位置
- 操作按钮无法支持完整战棋行动流程（武器选择、精神指令等）
- 菜单与光标交互未建立清晰数据流

技术环境：
- **引擎**: Godot 4.6
- **平台**: 移动端（触控优先）
- **渲染**: 等距投影（Isometric），cell_size = Vector2(128, 64)
- **UI**: CanvasLayer 确保不受相机变换影响

## Goals / Non-Goals

**Goals:**
- 实现 GridCursor 组件，在地图上渲染选中格子的视觉指示器
- 实现 ActionMenu 系统，支持完整单位行动流程（移动、攻击、精神指令、待机）
- 实现 MenuPanel 基础组件，作为可复用的移动端菜单容器
- 建立光标-菜单交互数据流：光标选中单位 → 显示行动菜单 → 选择行动 → 执行
- 为后续回合制战斗系统预留接口

**Non-Goals:**
- 本阶段不实现完整移动范围计算和路径显示
- 不实现攻击目标选择和武器选择子菜单
- 不实现精神指令具体效果
- 不实现回合制流程（玩家回合/敌方回合切换）
- 不实现战斗演出动画

## Decisions

### 决策 1: GridCursor 渲染方式

**选择**: 使用 Node2D + Line2D 绘制格子边框高亮

**理由**:
- 等距投影格子边框可通过 4 条 Line2D 实现，无需复杂 shader
- Node2D 便于位置更新（跟随 GridCoord 转换）
- 支持动画效果（边框颜色脉冲、宽度变化）

**备选方案**:
- TileMapLayer overlay → 拒绝：无法实现动态边框效果
- Shader Material → 拒绝：过度复杂，增加 GPU 负担
- Sprite2D → 拒绝：需要为每个格子尺寸制作预渲染素材

### 决策 2: ActionMenu 菜单项结构

**选择**: 使用枚举定义菜单项 + Array[Dictionary] 存储动态选项

**理由**:
- 枚举便于类型安全切换和信号传递
- Dictionary 存储菜单项元数据（名称、图标、可用状态）
- 支持运行时动态生成可用选项（如攻击仅在有目标时可用）

**备选方案**:
- 纯 Resource 文件 → 拒绝：难以动态生成选项
- 纯代码硬编码 → 满足当前阶段简单需求，后续可扩展

### 决策 3: 光标交互模式

**选择**: 光标始终跟随选中格子，单击确认 → 显示菜单

**理由**:
- 符合移动端触控直觉：先选择位置，再选择行动
- 光标移动与相机平移分离：相机可自由移动，光标锁定格子
- 单击确认时检查格子内容（单位/空格子）决定下一步

**备选方案**:
- 光标自由移动（不锁定格子）→ 拒绝：不符合网格战棋设计
- 双击确认 → 拒绝：移动端触控延迟过长

### 决策 4: MenuPanel 布局方式

**选择**: 使用 VBoxContainer + Button 列表布局

**理由**:
- VBoxContainer 自动垂直排列，适配移动端竖屏
- Button 继承自 Control，支持触控事件和最小尺寸约束（48x48）
- 便于扩展（后续可添加图标、分隔符）

**备选方案**:
- HBoxContainer → 拒绝：选项过多时横向布局不适合竖屏
- GridContainer → 拒绝：增加复杂度，当前阶段简单列表足够

### 决策 5: BattleCamera 信号扩展

**选择**: 新增 `cursor_moved(coord)` 和 `cursor_confirmed(coord)` 信号

**理由**:
- `grid_cell_selected` 已存在但语义模糊（选择 vs 确认）
- 新信号明确区分光标移动和确认操作
- GridCursor 可订阅 `cursor_moved` 更新位置，菜单订阅 `cursor_confirmed` 显示

**备选方案**:
- 保持单一信号 → 拒绝：无法区分移动和确认，交互逻辑混乱
- 用状态标志区分 → 拒绝：增加复杂度，信号语义更清晰

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 移动端触控精度影响格子选择 | 增大 GridCursor 边框宽度，支持触摸容差区域检测 |
| 光标动画帧率影响 | 使用 Tween 替代每帧更新，限制动画持续时间 |
| 菜单按钮过多占用屏幕空间 | 设计可收起的菜单面板，选择后自动隐藏 |
| 等距投影坐标转换精度 | 使用 GridCoord 工具类统一转换，避免直接计算 |
| 光标与相机交互冲突（拖拽时误触发选择） | 拖拽标志检测，拖拽结束后才允许确认选择 |

## Open Questions

1. **光标初始位置**: 场景加载时光标是否应该聚焦到地图中心，还是首个玩家单位位置？（倾向于地图中心，避免干扰）
2. **菜单显示时机**: 是否在选中空格子时也显示菜单（如"取消"选项），还是仅对单位格子显示？（倾向于仅单位格子显示）
3. **行动菜单选项可用状态**: 是否需要实时检查单位行动能力（如移动范围、攻击目标），还是全部显示？（倾向于全部显示，后续用 disabled 状态标记不可用）
